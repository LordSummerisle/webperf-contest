Faq

Est-ce que je dois suivre les réponses de cette FAQ ?

C'est préférable mais si vous avez de bons arguments, nous sommes prêts à discuter bien entendu.

J'ai le meilleur temps onload, startrender, nombre de requêtes, je vais gagner ?

Les juges étudieront les modifications et déciderons si vous gagnez. Les mesures sont utiles pour se faire une première idée du travail accompli.
Nous ne pouvons faire un classement seulement sur les chiffres car certaines optimisations sont trop complexes pour êtres intégrées et nous souhaitons donc aussi classer votre travail.

Pourquoi indiquer "utilisez les CSS3" alors qu'une partie des tests seront réalisés sur IE7 ?

Vous pouvez utiliser les CSS3 pour les navigateurs qui les supportent et avoir une version dégradée sous IE7. Par exemple pour des bords arrondis légers ou des ombres. Cela ne signifie pas que vous pouvez utiliser des CSS3 partout.

Pourquoi être si vague dans les règles et la faq ?

Ce concours comporte des judges, qui devront donc juger de votre travail. Si nous détaillons toutes les règles qui s'appliquent, cela crééera un cadre de travail trop restreint et de compétition il n y aura plus... Nous laissons donc libre les participants sur un ensemble de points, avec un but précis : optimiser une page web comme si vous étiez chez votre client et que vous lui proposiez des modifications. Elles ne doivent pas êtres trop compliquées à mettre en place et ne doive pas nuire à la qualité du site.

Est-ce que le mode repeat view de WPT sera utilisé ?

Oui.

Est-il possible de mettre tous les styles directement dans le index.html ?

Non. Sauf si vous me prouvez que c'est envisageable d'avoir un site web qui inclut toujours les mêmes styles en inline dans le html ? Posez vous toujours la question suivante : est-ce que je ferai ça pour mon site web ?
Si vous souhaitez vraiment faire ça, vous devriez réfléchir à comment faire pour que la page suivante inclue directement le fichier css externe dans le head ? Indiquez le dans votre gist comment vous voyez les choses et nous aviserons.

Je peux tout mettre en sprite ?

Non, vous feriez ça sur votre site e-commerce ? Comment feriez vous alors pour afficher un même produit dans deux contextes différents ? Vous chargeriez tous les sprites possibles ? Généralement pour les sprites on se limitera aux éléments d'interface.
Sauf si vous êtes facebook et que vous êtes capables de savoir quels sont les produits généralement affichés sur la même page. Même dans ce cas, le gain n'est pas significatif.

J'ai un problème avec la configuration serveur, quelque chose ne va pas...

La configuration est identique à celle du site de la FNAC, vous devrez probablement écrire des modifications dans le .htaccess.

Est-ce que je peux regrouper tous les appels ajax pour les disponibilités de produits ?

C'est une bonne idée et cela devrait être réalisable en production, oui.

Je peux utiliser font-face ?

Non. Font-face c'est vraiment génial pour nos portfolios de développeurs web mais en réalité l'utilisation sur un site grand public est vraiment complexe : différences de rendu, incompatibilités, formats, maintenance, hacks. Au fait, j'adore font-face sinon.

Est-ce que je peux générer avec javascript les paramètres UIDS, Origin semblables dans les liens ?

Oui, si vous conservez les bonnes valeurs.

Je peux supprimer l'IFRAME ?

Non, vous pouvez la charger différemment par contre.

Je peux supprimer des images, du code inutilisé ?

Oui si vous êtes certains que ça ne sert à rien.

Je dois conserver les champs hidden dans les formulaireS ?

Oui.

Je peux utiliser du CSS3 partout ?

Non. L'utilisations des CSS3 est autorisée mais si votre page ressemble à une feuille blanche sous IE7 alors il y a un problème.
Les CSS3 ne doivent être utilisés que pour des bords arrondis et des ombres difficiles à discerner (donc pas le menu du haut).

Les participations seront-elles rendues publiques ?

Oui à la fin du concours.

Est-il possible de travailler seulement en local et d'envoyer mes modifications à la fin ?

Oui. Aucun problème

Est-il vraiment utile de remplir son gist ?

Oui vous devez le faire si vous ne souhaitez pas être désavantagé. En plus on est un peu feigneants alors on cherchera pas longtemps...

Est-il nécessaire de fournir les fichiers sources ?

Si vous avez modifié beaucoup de code et que vos fichiers sont minifiés, il nous sera difficile de juger de votre travail (en plus on et feign..).
Alors s'il vous plait, créez un répertoire "source" sur votre FTP et mettez-y vos fichiers. Si vous liez les noms de fichiers dans votre gist alors vous serez notre chouchou.

Je peux changer mon uniqid ? (== je l'ai mis sur mon wall facebook)

Oui, contactez-nous.

Je peux mettre à jour mon url gist ? (== j'ai pas compris à l'inscription)

Oui, contactez-nous.

Je peux changer le nom du fichier index.html ?

Si vous ne souaitez pas apparaître dans le leaderboard, oui. Dans tous les cas, pour valider votre participation, vous devrez avant le 30 Novembre, avoir un fichier nommé index.html à la racine.

Je peux faire des redirections depuis index.html vers index-ie ou index-ff ?

Non. Tous les tests seront réalisés sur index.html et ce fichier doit renvoyer 200 OK. De plus vous ne devriez pas faire de redirection serveur en fonction du user agent. C'est une mauvaise pratique, une URL = un contenu identique pour tous.

Je peux faire des trucs cons comme charger une page blanche puis une iframe avec le contenu complet ? Ou bien un truc plus ouf genre charger un JPG avec toute l'interface du site puis repositionner les éléments dessus ?

Même Chuck Norris désaprouve ce genre de pratiques.
