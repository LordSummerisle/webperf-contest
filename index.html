Brief

Zeroload is organizing the first web performance contest, sponsored by YOTTAA.

We are a group of web performance junkies. We are planning a contest like the js1k competition but this time on the subject of WPO (Web Performance Optimization).

This is the first time a competition of this kind is organized (well, almost).

The contest will start the 25th October and end the 30th November.

How to enter the competition ?

When the contest starts, people will be able to download a Zip file from this website. The Zip will contain a complete static webpage (.html) with all the required assets (js, css, backgrounds, ads) for the page to work.

Each participants will receive a FTP account to put their files. Of course you won't be able to see other participant's urls before the end of the contest (also this removes ability to cheat with other people's optimized assets). The FTP accounts will be active for the time of the contest.

Participants will have 3 different subdomains, to maximize possibilities.

Each participant will provide their best magic skills to make the example webpage fast.

How to win ?

When the contest ends, each participant's url will be tested in multiple tools and ranked by theses categories :

  • Best YOTTAA score
  • Best sum of YSLOW, Pagespeed & Zoompf scores
  • Shortest loading time (start render + onload)
  • Best progressive rendering (humain eye check)
  • Smallest website size, tiniest number of requests
  • Innovation award / best idea

What to win ?

  • An IPAD, generously given by our sponsor, YOTTAA
  • Fame!
  • Performance skills
  • Self accomplishement
  • Want to give something ? A book, ticket for a conference ?

Rules

  • No cheating ! Displaying a big JPG for the complete website is cheating (even if you strip exif data, yes!)
  • No server side code (it is disabled)
  • Once page is loaded, all original features must still work
  • Ads and widgets must still be visible and usable
  • Accessibility must be preserved (also you can improve it)
  • SEO must be preserved (also you can improve it)

Then, you can use whatever technique you want : data uris, mhtml (external, inline), script defer ...
