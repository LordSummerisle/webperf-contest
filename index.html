Brief

Zeroload is organizing the first web performance contest, with the support of YOTTAA and Alwaysdata.

We are a group of web performance junkies organizing a contest like the js1k competition but this time on the subject of WPO (Web Performance Optimization).

This is the first time a competition of this kind is organized (well, almost).

The contest will start the 25th October and end the 30th November.

How to enter the competition ? (soon)

Download the files and register. The Zip contains a complete static webpage (.html) with all the required assets (js, css, backgrounds, ads) for the page to work.

You will then receive a temporary FTP account on a French host (provided by our sponsor, Alwaysdata) to put your files.

You will also be given a uniqid wich will be the directory name where to put your files.

All participant's directories are available on 3 different domains (entries.webperf-contest.com, s1.webperf-contest.com and s2.webperf-contest.com).

If you do not want to be copied, do not share your uniqid ! We won't be responsible for that.

How to win ?

When the contest ends, each participant's url will be tested in multiple tools and ranked by theses categories :

  • Best YOTTAA score
  • Best onload and start render
  • Innovation award / best idea

Both first and second access to the page will be taken into account.

All measures will be done from a typical DSL connection (1.5Mb/s, 50ms RTT), on the same server, with online analysis tools like yottaa.com, webpagetest.org (Paris location) and zoompf.com. You will be able to test your changes in the exact same conditions as us with, the FTP + HTTP account which will be given to you.

What to win ?

  • Books

  • Want to give something else ? A book, ticket for a conference, license for your software/solution ?

Rules

  • No cheating ! Displaying a big JPG for the complete website is cheating (even if you strip exif data, yes!)
  • No server side code (it is disabled)
  • User experience should not be degraded (but you are welcome to improve it), which is to say, for example:
    • Once page is loaded, all original features must still work
    • Ads, widgets or trackers must still be visible and usable
    • Accessibility must be preserved (also you can improve it)
    • SEO must be preserved (also you can improve it)

Then, you can use whatever technique you want : data uris, mhtml (external, inline), script defer ...

Htaccess files will work, so use it if you have to

You may deliver a page wich is not an exact pixel per pixel copy and do some arrangements (like replacing an image by some css or js code wich has a similar result), provided you respect the "user experience" rule. The judges will have the final word on what they think is acceptable or not.

we will probably be less strict for the "innovation award / best idea" criteria. If a typical product owner would refuse your change, you may be outside the rules. Ask if you're really unsure, we'll also contact you if there's a problem.
